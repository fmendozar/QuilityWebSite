How it works - Prototype

Digital Mockup showing Quility's main tasks

Task 1: Vocally affirming the completion of a task

This task gives the user an added sense of certainty that their task has been completed. We allow the user to store their vocal affirmations so that they will later on be able to check in on their past affirmations if they ever feel anxiety about a task.

"Hey Quility, I just turned off the stove"

In this screen, the user can tap specific buttons to: find ways to calm him down, check logs about all the actions recorded by Quility, check the “To do” list with all the tasks that the user commonly asks about. For this task the user can tap the microphone button, in order to record the realization of an action by using voice commands. We also have a switch at the top right corner of the screen, to give the user the ability of turning on/off the small camera (that can be put on a pair of glasses or clothes) at any moment.

"Hey Quility, I just turned off the stove"

In order to log an action by vocally affirming the realization of it, the user taps the microphone button and expresses what he wants to record.

"Sorry I didn't catch that. Please try again"

If Quility is not able to understand the user input, it pops up this error message to let the user know that nothing got logged and that it can be tried again. Otherwise, the user’s input is logged.

"Okay, I've logged you turning off the stove at 1:29pm"

Quility logged the completion of the action and makes it available in the logs screen.

Video about the action available to the user

The camera was able to record a video related to the action “turning off the stove”. The user presses the “watch video” button in the logs screen, and he gets redirected to this screen where he can play the video. He also has the possibility of deleting the video if he finds it appropriate.

Task 2: Calming down during an anxiety attack

This task was equally important to include because in our research anxiety was noted as one of the reasons OCD is so disruptive to people’s lives. Therefore, we want our solution to help our users directly combat this anxiety. By broadening our scope and changing our task to include multiple different calming methods we can help a larger variety of people. For example, within our task to calm anxiety we can allow our users to play calming music, call a friend, initiate a meditation phase, play nature noises, etc. The user can choose which calming method works for them.

Calming the user down. - Breath Exercise

With Quility, it’ll monitor your heart rate and also take voice commands in order to detect your level of anxiety and attempt to soothe it. It will take you to a screen that gives the user multiple options to reach mental tranquility to combat anxiety. The user can pick between musical soothing, breathing exercises, or meditation routines. These will take the user to a screen to provide the service for them.

Calming the user down. - Breath Exercise

The breathe exercise is our primary solution to calming the user down and consists of an in-and-out exercise meant for the user to think mindfully and slow down their thoughts.

Calming the user down. - Breath Exercise

The user will hold when breathing in, release while breathing out, and repeat to complete the exercise.

Calming the user down. - Breath Exercise

This is the screen representing the user releasing touch and breathing out.
